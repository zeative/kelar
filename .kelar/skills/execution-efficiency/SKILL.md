# KELAR Skill: Execution Efficiency (CLI-first + Script-first)

A senior developer doesn't edit 20 files manually. They write a script that does it in 2 seconds.

---

## THE EFFICIENCY DECISION TREE

Before starting ANY task, ask:

```
Can this be done more efficiently with a CLI tool or script
than by editing files manually?

YES → Write the script/command first, then execute
NO  → Proceed with manual implementation
```

**Signals that a script is better:**
- Same change in 3+ files
- Pattern-based transformation (rename, wrap, add import, etc.)
- Boilerplate generation (new module, new component, new CRUD)
- Data migration or transformation
- Any operation describable as "do X to all files that Y"

## APPROACH 1: CLI Commands

For operations that existing tools handle well:

### Text replacement across files:
```bash
# Replace across all TypeScript files
find src/ -name "*.ts" -exec sed -i 's/oldName/newName/g' {} +

# Replace with word boundaries (safer)
grep -rl "oldName" src/ | xargs sed -i 's/\boldName\b/newName/g'
```

### File operations:
```bash
# Rename files matching pattern
for f in src/**/*Service.ts; do mv "$f" "${f/Service/Repository}"; done

# Find all files with pattern
grep -rl "import.*OldModule" src/ --include="*.ts"
```

### Code analysis:
```bash
# Find unused exports (TypeScript)
npx ts-prune

# Find all usages of a function
grep -rn "functionName(" src/ --include="*.ts"

# Check for hardcoded values
grep -rn "localhost\|127\.0\.0\|hardcoded" src/ --include="*.ts"
```

## APPROACH 2: Generated Scripts

For complex transformations, AI writes a script and executes it:

### JavaScript/TypeScript (if project uses Node):
```javascript
// .kelar/scripts/batch-update.js
// Auto-generated by KELAR for: [task description]
const fs = require('fs')
const path = require('path')
const glob = require('glob')

const files = glob.sync('src/**/*.ts')
files.forEach(file => {
  let content = fs.readFileSync(file, 'utf8')
  // Apply transformation
  content = content.replace(/pattern/g, 'replacement')
  fs.writeFileSync(file, content)
  console.log(`Updated: ${file}`)
})
console.log(`Done: ${files.length} files processed`)
```

### Python (if available):
```python
# .kelar/scripts/batch-update.py
# Auto-generated by KELAR for: [task description]
import os, re
from pathlib import Path

for file in Path('src').rglob('*.ts'):
    content = file.read_text()
    updated = re.sub(r'pattern', 'replacement', content)
    if updated != content:
        file.write_text(updated)
        print(f"Updated: {file}")
```

### Bash (universal fallback):
```bash
#!/bin/bash
# .kelar/scripts/batch-update.sh
# Auto-generated by KELAR for: [task description]
find src/ -name "*.ts" | while read file; do
  sed -i 's/pattern/replacement/g' "$file"
  echo "Updated: $file"
done
```

## APPROACH 3: Code Generators

For boilerplate that follows a fixed pattern, AI creates a generator:

```javascript
// .kelar/scripts/generate-module.js
// Usage: node generate-module.js --name UserProfile --type service
const name = process.argv[3]
const type = process.argv[5]

const template = `
import { Injectable } from '@/core'
import { ${name}Repository } from './${name}.repository'

@Injectable()
export class ${name}Service {
  constructor(private readonly repo: ${name}Repository) {}
  
  async findById(id: string) {
    return this.repo.findById(id)
  }
}
`

require('fs').writeFileSync(
  `src/modules/${name.toLowerCase()}/${name}.service.ts`,
  template.trim()
)
console.log(`Generated: ${name}Service`)
```

## AFTER SCRIPT EXECUTION

Always verify:
```bash
# Show what changed
git diff --stat

# Spot check a few files
git diff src/[sample-file]
```

Output:
```
KELAR SCRIPT RESULT
───────────────────
Files modified: [N]
Sample check  : [show diff of 1-2 files]
Errors        : none / [list errors]

Looks good? (yes / rollback)
```

If rollback needed:
```bash
git restore src/  # undo all unstaged changes
```

## SCRIPT STORAGE

All generated scripts go in `.kelar/scripts/` at project root.
Named descriptively: `[date]-[action]-[target].js`
Example: `2026-03-20-rename-service-to-repository.js`

Keep scripts — they're documentation of what was done and can be reused or referenced.

---

## ADAPTIVE BEHAVIOR

**1-3 files, unique changes:** Manual edit, no script needed.
**3-10 files, same pattern:** CLI one-liner preferred.
**10+ files, complex pattern:** Generated script required.
**Recurring task (will happen again):** Code generator — invest once, reuse forever.
