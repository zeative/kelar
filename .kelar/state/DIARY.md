# KELAR Session Diary
> Auto-written at end of every session or on /kelar:pause.
> Your project's memory across time. Never delete entries.

---

## How This Works

At the end of each session (or when pausing), AI writes a brief entry here.
This gives you (and future AI sessions) a quick human-readable history of what happened.

---

## Entry Format

```markdown
## [Date] [Time] — Session #[N]

**Worked on:** [feature/fix/refactor name]
**Completed:** [what got done — bullet points]
**Decisions:** [any pattern/architecture decisions made]
**Blockers:** [anything that needs to be resolved]
**Debt logged:** [any tech debt added to .kelar/state/DEBT.md]
**Next session:** [exact next step to pick up from]
**Mood:** [honest assessment — "codebase clean" / "getting complex" / "needs cleanup soon"]
```
